#+TITLE: Lelpa
#+OPTIONS: toc:nil

Lelpa is a personal ELPA generated from the non-built-in packages used by
=~/.emacs.d=.

The recipes in =recipes/= were generated from:

- =~/.emacs.d/init.el=
- =~/.emacs.d/straight/build-cache.el=

See =PACKAGES.md= for the generated package set and the built-in packages that
were intentionally skipped for GNU Emacs 32.0.50.

* Local build

Install Docker first, then run:

#+begin_src sh
make bootstrap-package-build
make container-image
make container-build
#+end_src

The generated archive is written to =archives/=.

* Regenerate recipes

After changing =~/.emacs.d/init.el=, regenerate recipes from the current
straight cache:

#+begin_src sh
emacs -Q --batch -l scripts/generate-recipes.el \
  --eval '(setq myelpa-output-dir "recipes")' \
  --eval '(myelpa-generate)'
#+end_src

* Usage

After publishing =archives/= to your static host, add the archive to Emacs:

#+begin_src emacs-lisp
(add-to-list 'package-archives
             '("lelpa-snapshots" . "https://luciuschen.github.io/Lelpa/snapshots/"))
(add-to-list 'package-archives
             '("lelpa-releases" . "https://luciuschen.github.io/Lelpa/releases/"))
#+end_src

Replace =https://luciuschen.github.io/Lelpa/= if you publish the archive
elsewhere.

This repository is based on MyELPA.  The upstream documentation is available at
https://codeberg.org/tarsius/myelpa/wiki/.
